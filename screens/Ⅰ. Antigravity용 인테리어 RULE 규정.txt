Ⅰ. Antigravity용 인테리어 RULE 규정

(기준 · 판단 · 금지사항)

RULE 1. 정체성 규정 (Identity Rule)

우리는 “디자인 인테리어”가 아니다

우리는 시공 중심 인테리어다

핵심은 구조(목공) + 마감(타일)

👉 모든 콘텐츠는
**“누가 시공하는가 / 어떻게 시공하는가”**에 답해야 한다

RULE 2. 사람 우선 규정 (Human First Rule)

회사보다 **사람(형제)**이 먼저 보이게

상담자 = 시공자 = 책임자 구조 고정

“대표”라는 단어보다 **“직접 시공”**을 우선 사용

❌ 금지

팀 다수 / 협력업체 / 파트너 표현

“전문 디자이너가 설계합니다” 문구

RULE 3. 공정 통합 규정 (Process Integration Rule)

목공과 타일은 별도 서비스가 아님

항상 하나의 시스템으로 설명

필수 표현:

“목공과 타일을 함께 고려”

“구조와 마감을 동시에 설계”

❌ 금지

목공 페이지 / 타일 페이지 완전 분리

RULE 4. 히어로 메시지 규정 (Hero Rule)

감성 문구 사용 금지

추상적 가치 금지

✅ 허용 문구 구조:

도면을 이해하고
현장에 맞게 시공합니다


슬로건은 설명 보조용으로만 사용

RULE 5. 이미지 규정 (Image Rule)
사용 가능

실제 시공 현장

정리된 공정

구조·마감 접점이 보이는 컷

사용 금지

AI 렌더링

가구·소품 중심 감성 사진

사람 얼굴 클로즈업

Before/After 단독 이미지

👉 사람보다 공정이 먼저 보여야 함

RULE 6. 작업사례 규정 (Case Rule)

작업사례는 “결과 자랑”이 아님

판단을 돕는 설명 자료

필수 구조:

현장 조건

고민 포인트

공정 설계

목공 역할

타일 역할

마감 기준

❌ 금지

사진만 나열

“고급스럽게 완성” 같은 표현

RULE 7. 상담 규정 (Conversion Rule)

“견적”보다 “상담”을 우선

고객은 몰라도 시작 가능해야 함

필수 선택지:

목공 / 타일 / 둘 다 / 모르겠음

❌ 금지

긴 입력 폼

금액 유도 문구

RULE 8. CTA 규정 (Action Rule)

CTA는 항상 1개

문구 고정:

시공 상담하기

모바일에서는 하단 고정 필수

RULE 9. 디자인 규정 (Design Rule)

색상: 블랙 / 차콜 / 네이비

로고: UC 이니셜 중심

애니메이션 최소

영상 자동재생 금지

👉 “잘 꾸민 사이트” ❌
👉 “현장 잘하는 팀 사이트” ⭕

RULE 10. 검증 규정 (Validation Rule)

모든 페이지는 아래 질문을 통과해야 한다:

누가 시공하는지 보이는가?

목공과 타일이 왜 함께 필요한지 설명되는가?

상담 버튼이 즉시 보이는가?

하나라도 NO → 수정
